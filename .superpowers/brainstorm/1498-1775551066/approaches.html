마이네임이즈 — 아키텍처 접근 방식 3가지

비용 효율과 개발 속도 사이에서 최적점을 선택해주세요

A

정적 생성 (ISR) — 추천

고객 페이지를 빌드 시 정적 HTML로 생성. 수정 시 해당 페이지만 재생성 (revalidate).

✅ 트래픽 늘어도 서버 비용 거의 없음 (CDN 서빙)
✅ 페이지 로딩 속도 최고
✅ Supabase 무료 티어로 오래 버팀
⚠️ 고객 정보 수정 후 반영에 수초 딜레이
💰 월 비용 예상: Vercel 무료 + Supabase 무료 = $0~25
B

서버사이드 렌더링 (SSR)

매 요청마다 DB에서 데이터를 가져와 렌더링. 항상 최신 정보 보장.

✅ 수정 즉시 반영
✅ 구현이 단순
⚠️ 트래픽 많아질수록 서버 비용 증가
⚠️ Vercel 서버리스 함수 호출 비용 발생
💰 월 비용 예상: Vercel $20 + Supabase $25 = $45+
C

하이브리드 (ISR + 실시간)

기본 정보는 정적 생성, 조회수/링크 클릭 등 동적 데이터만 API로 실시간 처리.

✅ 비용 효율 + 실시간 분석 가능
✅ 향후 방문자 통계 기능 추가 용이
⚠️ 구현 복잡도 높음
⚠️ MVP에는 과할 수 있음
💰 월 비용 예상: Vercel 무료 + Supabase $0~25 = $0~35
공통 스택: Next.js 15 (App Router) + Supabase (PostgreSQL) + Vercel 배포
관리자 도구: 어떤 방식이든 별도 관리자 페이지에서 고객 명함 편집
